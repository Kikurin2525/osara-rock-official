# Claude Code 引き継ぎ

更新日: 2026-07-19

## 最初に把握すること

このリポジトリは株式会社オサラロックの公式サイトです。現在の本番サイトは https://osara-rock.com/ で、GitHub の `main` ブランチから Vercel に自動デプロイされます。

今回の目的は、現行の「3D空間を入口にした公式サイト」を実際に起動して確認し、Claude Code の視点でもう一段洗練することです。単に別案へ作り直すのではなく、現行より明確に良くなる改善を実装してください。公式情報、問い合わせ導線、検索性、モバイルでの使いやすさは維持します。

## ユーザーの要望

- 会社の公式HPとして、綺麗で恥ずかしくないインターフェースにする。
- 参考サイトは https://vibe.co.jp/ 。通常のコーポレートサイトではなく、サイト自体が空間体験になっている点を参考にする。
- 「レンタルスタジオ」「レンタルスペース」という見せ方から一歩進め、VR/バーチャルショールームとして体験できる公式HPにする。
- 朱色は使わず、真摯で落ち着いた印象にする。
- 強い言葉、断定的・攻撃的なコピーは避け、穏やかで誠実な表現にする。
- ブログは削除済み。復活させない。
- ユーザーの意見に迎合するだけでなく、事実に基づく率直な意見や、意外性のある提案も歓迎される。

## 重要なデザイン判断

- VIBE の素材、コード、文言、画面構成をコピーしない。参考にするのは「WebGL空間そのものをサイトの入口にする」という発想だけ。
- 全ページをゲームのような体験だけにすると、会社情報の発見性や問い合わせ率が落ちる。そのため、3Dショールームをファーストビューにしつつ、下部に通常の公式サイト情報を残している。
- 現在あるのは通常の店舗写真で、360度パノラマ画像ではない。実在しない店内を本物のように生成・表示しないこと。現状は「実写真を使ったインタラクティブ3Dショールーム」であり、真の360度VRではない。
- 将来、各店舗の正距円筒パノラマ画像が提供された場合は、実写360度ビューへ差し替える価値がある。
- ブランドの基調色は深い緑 `#2f6257`、チャコール、控えめなスチールブルー。朱色、派手な紫系グラデーション、一色だけに偏った配色は避ける。
- 公式サイトとしてのCTAと情報導線を、演出のために隠さない。

## 現在の実装

本番反映済みの最新コミットは `b44e8d0 Build interactive virtual showroom` です。

### 主要ファイル

- `src/components/virtual/VirtualShowroom.tsx`
  - React Three Fiber / Three.js / Drei による全幅の3Dショールーム。
  - DAYS、Tiarina、party、workspace の4空間を実写真パネルで表示。
  - 選択に合わせたカメラ移動、自動ツアー、前後移動、一時停止、全画面表示、3Dパネルの直接選択を実装。
  - WebGLやテクスチャの準備前は実写真モザイクを表示し、真っ白な画面を避ける。
  - `data-scene-ready` と `data-selected-space` はブラウザ検証にも使用している。
  - `preserveDrawingBuffer: true` はキャンバスの画素QAを安定させるために設定。性能とのトレードオフがあるため、変更する場合は実測する。
- `src/data/spaces.ts`
  - 4空間の画像、カテゴリ、説明、特徴、アクセント色を管理。
  - 未確認の店舗情報やURLを推測で追加しない。
- `src/app/page.tsx`
  - ファーストビューに `VirtualShowroom` を配置。
  - その下に実績数値、事業内容、実空間カード、運営支援、ニュース、問い合わせを残している。
  - 空間一覧のセクションIDは `spaces`。
- `src/components/layout/Header.tsx`
  - `スペース` が `/#spaces` へ移動する。
- `src/app/globals.css`
  - `.virtual-showroom:fullscreen` の全画面表示ルールを含む。
- `package.json`
  - `three@0.184.0`
  - `@react-three/fiber@9.6.1`
  - `@react-three/drei@10.7.7`

## 壊してはいけないこと

- ブログのナビゲーション、ページ、リダイレクト、設定を復活させない。
- 朱色をブランドカラーに戻さない。
- 強いコピーや否定的な比較表現に戻さない。
- 架空の店舗情報、営業情報、実績、リンクを追加しない。
- WebGL非対応時のフォールバックとアクセシビリティを失わない。
- デスクトップだけで成立するレイアウトにしない。320px幅を含むモバイルで、文字・ボタン・3D UIが重ならないようにする。
- ユーザーの未コミット変更がある場合は、勝手に取り消さない。
- `three` を安易に最新版へ上げない。`0.185.1` では現行R3Fとの組み合わせで `THREE.Clock` の非推奨警告が出たため `0.184.0` に固定している。

## 既知の課題

- 4枚の元画像は比較的大きい。実写感を損なわない範囲で、3D用の最適化画像を派生生成し、初期転送量を減らす余地がある。
- `npm audit --omit=dev` では Next.js 内包の PostCSS に moderate が2件出る。`npm audit fix --force` は Next.js を9.3.3へダウングレードするため適用しないこと。依存関係は個別に影響を確認して更新する。
- 現状は360度VRではない。360度パノラマ素材がない状態で、表現だけを「実写VR」と断定しない。
